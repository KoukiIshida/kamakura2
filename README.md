# 謎解きクイズ（ニックネーム分岐）

ニックネームを登録し、その内容に応じて「特定の1問」を出し分けできる謎解きクイズです。
**サーバー（PHP/MySQL）不要**。HTML/CSS/JS のみで動き、GitHub Pages で公開できます。

- **管理ページ（あなただけ使用）** … 問題と「ニックネーム分岐ルール」をGUIで設定し、`settings.json` を書き出す
- **参加者ページ（公開URL）** … `settings.json` を読み込み、ニックネーム登録→出題→答え合わせ

---

## ファイル構成

```
nickname-quiz/
├── index.html        … 参加者ページ
├── admin.html        … 管理ページ（あなた専用）
├── settings.json     … 設定データ（管理ページで書き出して差し替える）
├── css/
│   └── style.css
└── js/
    ├── common.js     … 共通処理（答えのハッシュ化・条件判定・読込）
    ├── app.js        … 参加者ページのロジック
    └── admin.js      … 管理ページのロジック
```

## 動かし方（ローカルで確認）

> ⚠️ HTMLファイルを**ダブルクリックで直接開くと動きません**（答えのハッシュ化と設定ファイル読込が、ブラウザのセキュリティ制限で動かないため）。必ず下のどちらかで開いてください。

### VS Code の「Live Server」拡張を使う（おすすめ）
1. VS Code に拡張機能「**Live Server**」をインストール
2. `nickname-quiz` フォルダを VS Code で開く
3. `index.html` を右クリック →「Open with Live Server」
4. 参加者ページ → `http://localhost:5500/index.html`
   管理ページ → `http://localhost:5500/admin.html`

## 使い方の流れ

1. **管理ページ（`admin.html`）** を開く
2. 「通常問題」「分岐問題」を追加し、問題文・答え・分岐ルールを設定する
   - 編集内容は**この端末のブラウザに自動保存**されます（下書き）
3. 「**settings.json を書き出す**」ボタンでファイルを保存
4. 書き出した `settings.json` を、このフォルダの `settings.json` に**上書き**する
5. **参加者ページ（`index.html`）** で動作確認

## 参加者に配る（GitHub Pages）

1. このフォルダ一式を GitHub のリポジトリにアップ（Push）する
2. リポジトリの **Settings → Pages** で公開を有効にする
3. 発行された公開URLを参加者に配る（QRコードにしてもOK）
   - 参加者ページ … `https://<ユーザー名>.github.io/<リポジトリ名>/`
   - 管理ページ … 同URLの末尾に `admin.html`（※あなただけが使う）
4. 設定を変えたいときは、管理ページで `settings.json` を書き出し直し、
   GitHub 上の `settings.json` を差し替えて Push する

> GitHubへのアップ（Push）はご自身で行ってください。

## 分岐ルールの仕組み

- 1つの分岐問題に、**ルールを複数**設定できます。
- ルールは**上から順に判定**し、**最初に当てはまったもの**を出題します。
- 1つのルール内の複数条件は **すべて満たす（AND）** 必要があります。
- どのルールにも当てはまらない人には、**デフォルト問題**を出します。

### 使える条件
| 条件 | 例 |
|------|-----|
| 文字数 | 4文字ちょうど／3文字以上／奇数 など |
| N文字目が〇 | 1文字目が「あ」／2文字目が「い」 |
| 前方一致 | 「あ」で始まる |
| 後方一致 | 「こ」で終わる |
| 含む | 「さ」を含む |
| 完全一致 | 「たろう」と一致 |

各条件には「**ではない**」チェックボックスがあり、条件を反転できます（例：「1文字目が『あ』**ではない**」「『さ』を**含まない**」）。

### 設定例（同梱の `settings.json`）
- ニックネームが **4文字** → Aの問題
- **3文字 かつ 1文字目が「あ」** → Bの問題
- **2文字目が「い」** → Cの問題
- どれにも当てはまらない → Dの問題（デフォルト）

## 答えを「参加者のニックネーム」にする

各問題には「**答えを『参加者が登録したニックネーム』にする**」というチェックボックスがあります。

- チェックを入れると、その問題は**出題内容（問題文・画像）がニックネーム条件で変わっても、正解はどれも“その参加者が登録したニックネーム自身”**になります。
- 例：分岐問題でA・B・C・Dの画像を出し分けつつ、答えはどの人も「自分のニックネーム」。
- このモードのときは、答えの入力欄は不要になるので自動で隠れます。
- ニックネームは参加者がその場で入力するため、`settings.json` には答えを保存しません（実行時にニックネームと照合）。大文字小文字・全角半角は自動で吸収して判定します。

---

## ニックネーム登録ルール（入力をはじく）

管理ページの「**ニックネーム登録ルール**」で、参加者が入力できるニックネームに制限をかけられます。条件を満たさないと、はじいて再入力を促します。

- **文字数の下限／上限**（上限を `0` にすると無制限）
- **使える文字種**：ひらがな／カタカナ／漢字／英字／数字（チェックした種類だけ許可。すべて未チェックなら制限なし）
  - チェックした種類**以外の文字**（記号・スペース・絵文字など）が含まれていると、はじかれます。
- スタート画面には「※ ひらがな・カタカナ／2〜8文字で入力してください」のような案内が自動表示されます。

### 呼び方（くん／ちゃん／さん／呼び捨て）を選んでもらう
- 「**呼び方（…）を参加者に選んでもらう**」をオンにすると、登録画面に4択（くん／ちゃん／さん／呼び捨て）が出ます（ニックネーム本体とは別）。
- 選んだ呼び方は、問題文・クリア画面で `{呼び名}`（例：たろうくん）として使えます。
- 答えの判定や分岐条件は、呼び方を含まない「ニックネーム本体」で行います。

## 問題文の差し込みタグ（ニックネームを文中に埋め込む）

問題文の中に「タグ」を書くと、参加者のニックネームに応じて自動で置き換わります。

| タグ | 置き換わる内容 | 例（ニックネーム「たろう」） |
|------|----------------|------------------------------|
| `{ニックネーム}` | ニックネーム全体 | たろう |
| `{呼び名}` | ニックネーム＋呼び方 | たろうくん |
| `{1文字目}` | 1文字目（数字を変更可。全角数字もOK） | た |
| `{2文字目}` | 2文字目 | ろ |
| `{最後の文字}` | 最後の1文字 | う |
| `{文字数}` | 文字数 | 3 |

- 例：問題文に `答えの1文字目は「{1文字目}」` と書く → 「たろう」さんには `答えの1文字目は「た」` と表示。
- 該当する文字が無い場合（例：3文字の人に `{5文字目}`）は空欄に置き換わります。

## 画像について

- 問題に画像を付けたいときは、管理ページの「画像（アップロード）」からファイルを選ぶだけです（**複数枚OK**。一度に複数選択も、追加で足すことも可能。並び順は ▲▼ で入れ替え）。
- 選んだ画像は `settings.json` の中に**埋め込まれて**書き出されます。
  - そのため、別途リポジトリに画像ファイルを置く必要はありません。
  - 参加者URLを配った**他端末でもそのまま表示**されます。
- ⚠️ 画像が大きいと `settings.json` が重くなり読み込みが遅くなります。**1枚あたり300KB以下**を目安にしてください（大きい場合は警告が出ます）。

## 答えの隠蔽について

- 1つの問題に**複数の想定解**を登録できます。管理ページの答え欄に **カンマ「,」区切り**で入力すると（例：`さくら,サクラ,桜`）、そのどれか1つに一致すれば正解になります。
- 答えは `settings.json` に **SHA-256 ハッシュ**で保存され、原文は含まれません。
- 参加者がソース表示や開発者ツールを見ても、正解の文字はわかりません。
- 答えの照合はブラウザ内で「入力のハッシュ」と「保存されたハッシュ」を比較して行います。
- ※ 静的サイトのため理論上は総当たりの余地があります。より強固にしたい場合は、
  将来 nazoxnazo（PHP + MySQL）へ移行し、サーバー側照合に切り替えてください。
